# i18n — Translation Audit & Sync

Standalone workflow for auditing, fixing, and synchronizing translations.

## Steps

1. **Detect stack and translation system**:
   // turbo
   - **Laravel**: check `lang/` or `resources/lang/` for PHP/JSON translation files
   - **React/Inertia**: check for `t()` usage and JSON translation files
   - **Node.js**: check for `i18next` or similar
   - Identify active locales (e.g., `en`, `pt_BR`)

2. **Audit** — scan for hardcoded user-facing strings:
   - Route to `i18n-manager` skill for comprehensive audit
   - Or run quick scan:
     - **PHP**: grep for strings in Blade/controllers not wrapped in `__()`
     - **TSX/JSX**: grep for literal text not wrapped in `t()`
     - **Validation**: check Form Requests for hardcoded messages

3. **Report**:

   ```markdown
   ## 🌐 i18n Audit Results

   ### Hardcoded strings found

   | File | Line | String | Suggested Key |
   | ---- | ---- | ------ | ------------- |
   | ...  | ...  | ...    | ...           |

   ### Translation gaps (key exists in one locale but not another)

   | Key | Present In | Missing From |
   | --- | ---------- | ------------ |
   | ... | ...        | ...          |

   ### Orphaned translations (key in file but unused in code)

   - `key.name` in `pt_BR.json`

   ### Summary

   - Hardcoded strings: X
   - Missing translations: Y
   - Orphaned keys: Z
   ```

4. **Fix** (if user approves):
   - Replace hardcoded strings with translation helpers
   - Add missing translation keys to all locale files
   - Optionally remove orphaned keys
   - Route to `i18n-manager` skill for execution

5. **Verify**:
   // turbo
   - Re-run audit to confirm zero hardcoded strings
   - Verify all locales have matching keys
   - Run lint to ensure clean code

> **Post-Code Gates**: After fixing, also run Security scan (see `AGENT_FLOW.md` §3).
